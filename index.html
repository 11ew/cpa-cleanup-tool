CPA 清理工具

纯静态独立清理工具。页面本身不改 `cli-proxy-api.exe`，也不改 `static/management.html`。 删除前会先把原始 auth 文件备份到当前浏览器本地回收站，再调用现有管理 API 删除，尽量降低误删风险。
当前仅支持 Codex 清理 不修改上游二进制 不修改现有管理页 回收站保存的是原始凭证 JSON，请只在受信任浏览器使用
思路参考开源项目 qcmuu/cpa-codex-cleanup， 当前页面为独立整理后的实现。
目前优先解决 Codex 清理场景；如果后续有时间，或者实际反馈不错，再考虑继续扩展其他模型。

连接与策略

回收站初始化中
管理密码只保存在当前页面内存，不会自动写入本地存储。回收站会把删除前的原始 JSON 保存到当前浏览器 IndexedDB，用于恢复。
支持填写 `management.html`、`/v0/management`、`/auth-files` 或根地址，工具会自动转换成正确的接口地址。
建议工作流：先点“先检查一遍”预览，再点“备份后删除”。这样即使误删，也能从本地回收站直接恢复。
回收站里保存的是原始 auth JSON，通常包含敏感凭证。请仅在你自己的机器、受信任浏览器和受信任用户环境里使用。
