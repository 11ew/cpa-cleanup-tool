# CPA Cleanup Tool

一个给 CLIProxyAPI / CPA 管理接口使用的独立清理工具。

它的目标很明确：

- 不修改 `cli-proxy-api.exe`
- 不修改 `static/management.html`
- 尽量不受上游升级影响
- 删除前先放进本地回收站，降低误删风险

## 致谢与来源说明

这个工具在思路上参考了开源项目：

- `qcmuu/cpa-codex-cleanup`
- 项目地址：<https://github.com/qcmuu/cpa-codex-cleanup>

当前仓库是在参考其清理思路后，按“纯静态独立工具 + 本地回收站保护 + 尽量不改上游”的方向重新整理和改写的实现。

这个工具是一个纯静态页面，浏览器打开就能用。  
它会直接调用现有管理接口完成扫描、试用检查、删除和恢复。

## 当前范围

目前这个工具只面向 Codex 相关 auth 的清理场景，尤其是 Codex 账号的异常筛查和试用检查。

暂时没有把其他模型或其他 provider 的主动探测做成通用方案。

如果后续有时间，或者实际使用反馈不错，再考虑继续扩展其他模型的清理支持。

## 适合谁

- 你已经在用 CLIProxyAPI
- 你不想改上游管理页
- 你需要定期清理失效或异常的 auth 文件
- 你希望删之前先有一层“回收站”保护

## 它能做什么

- 读取现有 auth 列表
- 根据现成状态信息找出明显异常项
- 对可疑的 Codex 账号做一次实际试用检查
- 删除前先备份原始 JSON 到浏览器本地回收站
- 支持从回收站恢复
- 支持导出和导入回收站

## 用到的管理接口

- `GET /v0/management/auth-files`
- `GET /v0/management/auth-files/download`
- `POST /v0/management/api-call`
- `DELETE /v0/management/auth-files`
- `POST /v0/management/auth-files`

## 快速开始

1. 启动你的 CLIProxyAPI，并确保管理接口可访问。
2. 用浏览器打开 `index.html`。
3. 填写“管理地址”和“管理密码”。
4. 先点“先检查一遍”，预览候选。
5. 确认无误后，再点“备份后删除”。
6. 如果误删，可在“本地回收站”中恢复。

## 界面说明

- 管理地址：CLIProxyAPI 的管理页面或管理接口地址，工具会自动转换成正确的 API 根路径。
- 管理密码：访问 CPA 管理接口用的密码，不是 OpenAI token。
- 实际试用检查：拿可疑账号去请求一次上游接口，判断它现在还能不能用。
- 明确异常：根据现成状态文本已经能判断出有问题的项目。
- 疑似 401：看起来像“未授权 / token 失效”的候选。
- 备份后删除：先把原始 JSON 保存到本地回收站，再调用删除接口。

## 回收站怎么工作

- 删除前，工具会先下载原始 auth JSON
- 然后保存到当前浏览器的 IndexedDB
- 删除成功后，可通过“恢复”重新上传回管理接口

## 安全提示

- 管理密码只保存在当前页面内存，不自动持久化
- 回收站里保存的是原始 auth JSON，通常包含敏感凭证
- 请只在受信任机器、受信任浏览器环境中使用
- 如果你清空浏览器站点数据、使用隐身模式、或更换浏览器，回收站数据可能丢失

## 限制

- 这是前端本地执行，不是服务端后台任务
- 页面关闭、刷新或浏览器崩溃时，当前执行会中断
- 回收站依赖浏览器 IndexedDB；如果浏览器禁用 IndexedDB，则会禁止删除操作
